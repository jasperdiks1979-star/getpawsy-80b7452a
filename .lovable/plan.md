
# Commercial Reality Sprint — Execution Plan

CIE proved tracking is healthy and the 0 purchases are real. Sprint goal: in 48h move the needle on **session quality, expectation-match, PDP→ATC, checkout-start, and paid orders**, plus give Founder one screen that shows it. Below is the 4-wave plan; each wave is independently shippable.

## Wave 1 — Truth & Redundancy (foundation, ~today)

1. **GA4 Measurement Protocol fallback in `stripe-webhook`** (`checkout.session.completed` + `payment_intent.succeeded`). Fires `purchase` server-side with `transaction_id = order.id`, `value`, `currency`, `items`, plus `client_id` recovered from `session.metadata.ga_client_id`. Idempotent via `ga4_mp_sent` flag on `orders`. Closes the refresh / clearCart blind spot for good.
2. Patch `create-checkout` to stamp `ga_client_id`, `ga_session_id`, `utm_*`, `referrer` into `session.metadata` so MP can reconstruct attribution.
3. New secrets needed (request via `add_secret`): `GA4_MEASUREMENT_ID`, `GA4_API_SECRET`. Without them MP no-ops, never blocks checkout.

## Wave 2 — Conversion Reality Engine (the analyzer, ~tomorrow AM)

New edge function `conversion-reality-analyzer` runs hourly. Inputs: `cie_sessions`, `cie_journey_steps`, `checkout_funnel_events`, `orders`, `pinterest_funnel_events`, `gi_traffic_sessions`. Produces:

- **Real funnel** segmented by source / medium / campaign / country / device / product_id.
- **Traffic quality score** per session (0-100) using: engaged ≥10s, scroll>25%, ≥2 page_views, not flagged bot.
- **Mismatch rate**: % of sessions whose entry product != viewed PDP within 30s.
- **PDP conversion score** per product (view_item → add_to_cart).
- **Checkout-start rate** (ATC → begin_checkout).
- **Top 10 leaking products** with the specific drop-off step.
- **Concrete fixes** generated by mapping leak-pattern → playbook (e.g. ATC>0 / checkout=0 → "variant friction or price shock").

Writes to new tables `conversion_reality_runs`, `conversion_reality_products`, `conversion_reality_segments`.

## Wave 3 — Founder Dashboard `/admin/conversion-reality` (~tomorrow PM)

Single screen, mobile-readable, reading Wave 2 tables:

- Hero: 5 KPI tiles (Sessions, Traffic Quality, PDP Conv, Checkout-Start, Paid Orders) with 24h vs 7d delta.
- Waterfall funnel with drop-off % per step.
- Segment table (source × device) with conversion + quality.
- Top leaking products with the concrete fix.
- Live incident feed (Wave 4).
- "Run now" button → invokes analyzer.

## Wave 4 — Auto-Incidents & AI Gating (~day 2)

Database trigger + analyzer rules open `cie_incidents` rows automatically when:

| Rule | Threshold |
|---|---|
| ATC high, checkout=0 | ≥5 ATC / 0 begin_checkout in 6h |
| Median session duration <3s | per source/24h |
| Paid traffic arriving as `direct` | >20% of paid clicks |
| Creative ↔ product mismatch | PRE relevance score <80 on a published pin |
| Clicks high, checkout-start=0 | ≥30 PDP views / 0 ATC in 12h |

AI gate: extend `cie_settings.ai_training_min_confidence` check so AGD/PEI/AEE skip any product whose `conversion_reality_products.confidence < 90`. Already a CIE invariant — this wave just plugs the new metric into it.
