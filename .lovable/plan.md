## Phase 2 — PCIE2 as the sole publisher
1. New edge function `pcie2-publisher` — only function allowed to call `/v5/pins`. Reads from new `pcie2_publish_queue` (status=`approved`, gate=`passed`).
2. Mandatory in-order gates (each writes to `pcie2_pipeline_trace`):
   - `classifier` → `pcie2_product_understanding` row with confidence ≥ 0.85.
   - `headline_intel` → headline picked from `pcie2_headline_library` (no template fallback). Reject if headline contains banned phrases or category mismatch tokens.
   - `hook_intel` → hook from `pcie2_hook_library` matching product class.
   - `creative_intel` → image generated by PCIE2, never a raw CJ supplier URL. Hard-block when source URL host is on CJ/supplier list or when `media_audit.status ∈ {BLOCKED, REVIEW}`.
   - `similarity_gate` → dHash compared against last N PCIE2 pins for the product; reject ≥ 0.70.
   - `quality_gate` → multi-axis Gemini score ≥ 0.85; deterministic checks (size, dims, contrast).
   - `integrity_guard` → existing `verifyPinIntegrity` (slug/UTM/species match).
3. Publish step asserts `meta.pipeline === 'pcie2'`, `trace_id`, and all gate `status='pass'` before POST.
4. Single feature flag: `pcie2_publish_enabled` (default false until E2E passes).

## Phase 3 — Pipeline Trace
New table `pcie2_pipeline_trace`:
- `id`, `pin_queue_id`, `product_id`, `trace_id`, `module`, `module_version`, `input_hash`, `output_hash`, `status` (pass/fail/skip), `reason`, `evidence` (jsonb), `duration_ms`, `created_at`.
Every gate writes one row. Publisher writes a final `published` row containing the Pinterest `pin_id`, board, destination URL, and the full ordered module chain. Admin route `/admin/pinterest-pipeline-trace/:pinId` renders the full chain + module versions + evidence.

## Phase 4 — Hard-block enforcement
Centralised reject reasons (`pcie2_reject_reason` enum):
- `irrelevant_headline`, `cj_supplier_image`, `duplicate_creative`, `low_quality`, `wrong_category`, `integrity_failed`.
Any reject sets `pcie2_publish_queue.status='rejected'` with reason + trace. Publisher cannot read rejected rows.
